# 🎵 大發音樂管理系統 (Dafa Music Management System)

## 📋 使用者故事 (User Stories)

### 1. 學生管理與課程自動化
* **身為櫃檯人員**，我想要在學生資料頁面編輯姓名、電話與地址，以便於維護最新聯絡資訊。
* **身為櫃檯人員**，我想要在學生頁面直接選購課程，並由系統根據「起始日」與「堂數」**自動在行事曆生成未來週數的課程**，以便於省去手動排課的時間。

### 2. 老師管理與連動
* **身為管理員**，我想要建立老師資料庫並與學生課程綁定，以便於在行事曆上同時追蹤師生雙方的行程。

### 3. 智慧行事曆 (Calendar View)
* **身為使用者**，我想要一個類似 Google Calendar 的介面，可以自由切換**日/周/月**檢視。
* **身為工作人員**，我想要在日曆上直接看見「課程名稱/學生/老師」，並能透過**「請假」與「調課」**按鈕快速異動行程。

### 4. 課程清單設定
* **身為管理員**，我想要自定義課程選項（包含名稱、總堂數、收費價格），以便於在學生購課時能直接下拉選取。

### 5. 極簡簽到系統
* **身為櫃檯人員**，我想要一個只顯示今日學生的簡潔頁面，並在點擊學生後僅顯示**「簽到」與「請假」**兩個大按鈕，以便於在繁忙時快速操作。

---

## 🛠 功能架構圖 (System Architecture)



| 功能模組 | 關鍵欄位 / 功能 |
| :--- | :--- |
| **學生資料庫** | 姓名、電話、地址、購課紀錄、自動排課邏輯 |
| **老師資料庫** | 基本資料、授課學生清單、授課時段連動 |
| **行事曆** | 日/周/月切換、課程卡片(師生資訊)、請假/調課彈窗 |
| **課程列表** | 課程名稱、單位堂數 (e.g. 8堂)、收費標準 |
| **簽到頁面** | 今日名單篩選、極簡化「簽到/請假」二選一介面 |

---

## 💡 實作建議 (Technical Notes)
1. **排課邏輯**：系統應撰寫一個 `Loop`，根據學生選購的 `Total_Lessons`，從 `Start_Date` 開始每隔 7 天寫入一筆資料至 `Calendar_Events` 資料表。
2. **簽到狀態**：簽到頁面的操作應直接更新行事曆該堂課的狀態碼（例如：0-未到, 1-已簽到, 2-請假）。
